Objectives

In this lab we describe how to

  • Install MiKTeX
  • Install and configure Eclipse LaTeX plugin (Texlipse)
  • Customise installation to generate WIT Programming slide decks
  • Test the system by rebuilding an existing slide deck

MikTeX Setup

Install MikTeX via ProTeXt

Here are the instructions to install MikTeX on a Windows XP machine

  • Download ProTeXt from here
  • Downloaded file should be ProTeXt-3.1.3-060313.exe or similar
  • Extract into a temporary directory (double click)
    • Run Setup
    • This will launch a proTeXt window (Figure 1)
    • In this window are provided options to install MikTeX and TeXStudio
      • Do not install TeXStudio
      • Install MikTeX (accepting defaults)

Eclipse

Install Eclipse TeXlipse plugin

You can create your slide decks from witin Eclipse using the Texlipse plugin.

Here are the steps necessary to install and configure Texlipse:

  • Launch Eclipse
  • Open Help | Install New Software
  • A window named Install opens (Figure 1)
    • Insert the path to the texlipse repo in the input control named Work with:
      • http://texlipse.sourceforge.net
    • Press the Add button
    • This launches the Add Repository window (Figure 2)
    • Type, for example, TeXlipse in the Name control
    • The location control should show http://texlipse.sourceforge.net
    • Press OK
    • Accept defaults as you progress through the install wizard (Figure 3)
    • You should finalize the install by relaunching Eclipse

Configure Texlipse

  • Open Windows | Preferences | Texlipse | Builder Settings (Figure 4)
  • Browse to Bin directory of TeX distribution (Figure 5)
    • All the Builder Settings window should be populated as shown in Figure 6
    • Press Apply
    • Press OK

Create LaTeX Document

Test drive Texlipse

Once TeXlipse has been installed and configured test the system as follows:

  • Launch Eclipse
  • Switch to the appropriate workspace

  • Execute File | New | Other | Texlipse | LaTeX Project to launch LaTeX Project Wizard (Figure 1)

  • Press the Next button

  • In the next pane, Latex Project Wizard, name the project, lecture01 and

  • Change the Output Format from the default dvi to pdf (Figure 2)

    • The Build commands should read pdflatex.exe

  • Scroll to Beamer in the Templates window

    • Beamer (LaTeX) is a class for creating presentation slides

  • Press Next to progress to the following page of the LaTeX Project Wizard (Figure 3)

  • Change from the default name document.pdf to lecture01.pdf (Figures 4)

    • We are giving the generated pdf document the same name as the project for convenience. This decision is a matter, however, for the author.

  • Press Finish button

  • A window titled Open Associated Perspective opens: (Figure 5)
    • Tick Remember my decision
    • Press Yes
  • lecture01.pdf has now been generated using a Beamer template (Figure 6)

Configuring to use external pdf reader

The following configuration change makes it easy to open pdf files generated by Texlipse using an external reader such as Adobe:

  • Open Eclipse Preferences
  • Type the prompt file in the Preferences window (Figure 7)
  • Select File Associations

  • Select *.pdf in the File types window

    • In the Associated editors window you should see PDF Viewer (Figure 8)
      • This is the embedded viewer that facilitates viewing pdf docs within Eclipse
    • Press the Add button and browse to an alternative viewer such as Adobe (Figure 9)
      • Repeat for other viewers you may wish to use

  • In the lecture01 project window select lecture01.pdf and right-click to open the context menu

    • Select PDF Viewer : the document opens within Eclipse (Figure 10)
    • Select AcroRd32 : the document opens in the external Adobe reader (Figure 11)

The project name lecture01 and the associated document name lecture01.pdf have been arbitrarily chosen.

Test

Import sample slide deck project

  • Expand the archive file lecture_11.zip
  • Launch Eclipse
    • Menu command: File | Import | Import Projects into Workspace
    • Press the Next button
    • In Import Projects window browse to the location of lecture_11 folder
      • You may wish to tick Copy projects into workspace
    • Finish

Use imported project to test the system

  • Open the project (lecture_11) in the Project Explorer
  • Delete lecture_11.pdf
  • Open the file lecture_11.tex and touch the file (without making any material change)
  • Then save the file: this forces a rebuild and regeneration of the pdf slide deck.
  • Open the pdf using both embedded and external viewers and check it has been correctly regenerated.
